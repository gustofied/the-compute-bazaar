The Compute Bazaar: Tinkering with Compute Markets

17 June 2026 · local dashboard notebook

This local page is the working interface for the project. The same file should later move into the AdamSioud exemplar flow, with the static JSON source swapped for a public S3 or CloudFront prefix.

Compute is not scarce in one clean place. It is scattered through provider APIs, regional machines, private terms, spot-like availability, and vague promises of performance. The first Compute Bazaar is therefore not an exchange. It is a measuring instrument: a small public price tape for watching GPU capacity become legible.

I The Substrate

The substrate is boring on purpose. Bronze keeps raw provider evidence. Silver turns each provider into one shared offer schema. Gold builds product-facing facts: listings, provider floors, index values, regions, and the constituents supporting those values.

II The Markets

The first market view is deliberately simple: for each GPU product, find the available floor in dollars per GPU-hour. The point is not a final methodology. The point is a reliable path from raw observation to an auditable number.

IV Frictions and Futures

The public essay should start with static JSON snapshots because static pages are robust, cheap, and safe. Later, a small backend can consume AutoMQ and expose a narrow live feed without putting broker credentials in the browser.
